# Product Brief — AI Center (ai-center)

## 1. Executive Summary

**AI Center（ai-center）** 是「小杏仁」产品体系中的统一 AI 能力中心，为 Web、小程序及 Java 后端提供标准化的 AI 分析服务。

小杏仁的核心理念是：**用户不需要组织想法，只需要把想法“放下”**。AI Center 通过 AI 自动理解、分类和分析用户的自然语言输入，帮助用户减轻认知与心理负担，让每一个一闪而过的念头都能被合理记录和利用。

AI Center 同时也是一个通用 AI 基础设施，未来将服务于小杏仁以外的其他应用。

---

## 2. Problem Statement

### 核心问题

目标用户通常拥有大量想法，但这些想法往往：

* 在产生的一刻显得重要
* 仅被简单记录为一行笔记
* 随后被遗忘或无法转化为行动

### 典型场景

**场景一：碎片化想法被浪费**
在工作中突然想到「需要研究 Python 的某个技术点」，只随手记下一行笔记，由于缺乏整理与提醒机制，最终被长期搁置。

**场景二：空闲却无从下手**
忙碌时积累了大量想做的事情，而真正空闲时却不知道该从哪一件开始。

### 现有方案的不足

* 传统笔记工具要求用户**先想清楚如何分类**
* 记录本身就构成认知负担
* 无法将“想法”自然转化为“行动”

### 不解决的后果

* 时间被持续浪费
* 与善用 AI 提升效率的人逐渐拉开差距

---

## 3. Target Audience

### 主要用户

* 18–25 岁学生
* 25–35 岁职场人

### 用户特征

* 想法多、节奏快
* 不擅长系统化整理
* 希望工具足够简单、足够聪明

### 当前使用工具

* 系统备忘录
* Notion
* 零散笔记工具

---

## 4. Solution Overview

### 当前 MVP（本期重点）

AI Center 当前专注于 **智能分类能力**：

* 用户输入一段自然语言文本
* AI 自动识别该内容属于：

  * 记忆
  * 计划
  * 目标
  * 决策
  * 行动 / 待办 等类型
* 使用 **通义千问（Qwen 系列）模型**
* 以 **后端 API 服务** 形式提供能力

AI Center 本身不面向终端用户，仅服务于后端系统。

### 未来方向（非本期）

* 多模态记忆增强（图片 / 声音 / 立体联想）
* 决策支持与思维模型
* 智能行动推荐系统

## 5. Business Objectives

### 短期目标（3–6 个月）

* 小杏仁 MVP 上线
* 验证“放下想法 + AI 自动整理”的核心价值
* 积累真实用户数据和反馈

### 成功指标

* API 稳定可用
* 分类准确率 ≥ **90%**
* 用户数量 ≥ **100 人**

### 长期目标

* 打造个人效率工具体系
* 走 **付费商业化** 路线

### 项目意义

这是作者作为程序员以来，第一个完整跑通的、体系化的个人项目闭环。

---

## 6. Scope

### In Scope（本期必须完成）

* 智能分类 API
* 演化分析与复盘总结 API
* LangGraph 工作流引擎
* 安全与调用控制：

  * 仅允许 Java 后端调用
  * 前端不可直连
  * 用户 Token + Client Key / Secret 校验
* Nacos 服务注册
* Redis 缓存（必要时）
* 日志级别数据记录（不做业务数据库）
* Docker 部署
* GitHub Actions 自动化部署

### Out of Scope（本期不做）

* 决策支持与思维模型
* 智能推荐系统
* 移动端 App
* 独立监控系统
* 业务数据库（由 Java 后端负责）

## 8. Constraints & Assumptions

### 约束条件

* 单人开发，时间与精力有限
* 控制 LLM API 调用成本

### 假设条件

* 通义千问 API 稳定可用
* Java 后端可提供完整用户与鉴权信息
* Nacos 与 Redis 基础设施已就绪
* 100 位真实用户足以验证产品价值

---

## 9. Success Criteria

* 分类结果准确、稳定、可解释
* API 对业务系统稳定可靠
* 用户能明显感知记录负担降低、想法利用率提升
